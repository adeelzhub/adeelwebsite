1.6 Describing access issues that need taking into account - describe issues that people with disabilities may face when using webpages and how we can overcome them

Web accessibility refers to the extent to which a website is usable by its intended users taking into account their differing capabilities.[1] To make WebPages more accessible there are few things which should be taken into account. Visually impaired person may not be able to see an image displayed but a text description can help them to understand what is being displayed. If the webpage is not navigatable without mouse then it can be a problem for someone who cannot use mouse to navigate. A webpage should be structured in way that it can be navigated logically using tab key. Some people find animation or moving object really distractive. There should be an option to stop or pause animations. Some forms expire after a certain time. Screen readers take longer to read them. User should be informed when the form is going to expire and they should have an option to extend the time.
[1] https://www.boia.org/blog/the-most-common-web-accessibility-issues-to-avoid
